# UX Audit — Guitar Tutor
**Date:** 2026-02-21
**Persona:** Beginner guitar student, first time opening the app

---

## Executive Summary — Top 5 Highest-Impact Issues

1. **Jargon wall on first launch** — Terms like "Interval," "Mode," "Inversion," "Sus4," "vii°" are never explained anywhere. A beginner lands in the app and immediately faces unfamiliar vocabulary with no onboarding, glossary, or tooltips.

2. **Unclear tab purpose and no learning path** — Five tabs (Chords, Scales, Progressions, Triads, Arpeggios) with icon-only labels. A beginner won't know what "Triads" means, why it's separate from "Chords," or in what order to learn anything.

3. **Header controls are cryptic symbols** — LH, C, ♯/♭, ☾ have no labels. A beginner won't understand what "LH" stands for, what "C" with +/− does, or why any of these controls exist. The capo control is a known issue (TODO entry) and the most egregious case.

4. **Progressions tab requires theory fluency** — Roman numerals (I, ii, iii, IV, V, vi, vii°), the Circle of Fifths, and "diatonic chords" all presuppose music theory knowledge that a beginner doesn't have. There is no simple "start here" path.

5. **Fretboard dots have no legend** — Every screen shows a fretboard with colored dots and no explanation of what the colors represent. Root = orange, 3rd = blue, 5th = green, 7th = pink — none of this is communicated to the user.

## Per-Tab Findings

### App Shell & Header (`App.tsx`)

- **LH button** (`headerRight`, lines 54–62): Shows text "LH" with no tooltip, icon, or label. An accessibility label exists for screen readers, but sighted beginners get nothing.
- **Capo control** (`headerRight`, lines 63–83): Displays `− C +` and a value like "C0". "C" could mean C major to a musician, or nothing at all to a beginner. The +/− tap targets are tiny (6 px padding). No visual affordance that these are interactive. No explanation of what a capo does.
- **♯/♭ toggle** (`headerRight`, lines 84–93): Two music symbols with no label. A non-musician won't recognise them. The effect (switching C# ↔ Db) is invisible on a blank fretboard.
- **Theme toggle** (`headerRight`, lines 94–103): Sun/moon icons — passable. Most users understand this pattern.
- **Tab icons** (`Tab.Navigator`, lines 107–163): Icon-only tabs. No text labels. The icons are generic music symbols that don't convey the tab's function. "Triads," "Progressions," and "Arpeggios" are all terms a beginner may never have encountered.
- **First-launch state**: App opens to the Chords tab with a blank fretboard, no hint text, no onboarding, and no "start here" signal. A user who taps LH, C, or ♯/♭ will see no visible effect until they've already selected a root and chord type.

---

### Chords Screen (`ChordsScreen.tsx`)

**Strengths:**
- Predictable layout: root → type → display → fretboard.
- Hint text "Tap a root to change voicings" is useful.
- Voicing counter ("C Major (1/5)") communicates that alternatives exist.

**Issues:**
- **Chord type labels**: "Sus4", "Min7b5", "Maj7", "Dim7", "9th" — all music theory jargon with no explanation.
- **Display toggle modes**: "Finger", "Interval", "Note" — no description of what each mode shows.
  - Finger mode: shows numbers 1–4 (index through pinky), "O" (open), "R" (root). Unexplained.
  - Interval mode: shows "R", "3", "5", "7", "b3", "b7" etc. Unexplained.
  - Note mode: most beginner-friendly, least explained.
- **Fretboard colours**: Root = orange, 3rd = blue, 5th = green, 7th = pink, others = gold. None of this is communicated anywhere.
- **Multiple voicings**: No guidance on which voicing to start with or how they differ.

---

### Scales Screen (`ScalesScreen.tsx`)

**Strengths:**
- Box position picker shows fret range ("Box 1 (1–4)"), which is helpful.
- Scale types are more self-evident than chord types.

**Issues:**
- **Scale type labels**: "Nat. Minor" abbreviation is opaque. "Harm. Minor" and "Melodic Minor" are unexplained.
- **Mode picker**: Ionian, Dorian, Phrygian, Lydian, Mixolydian, Aeolian, Locrian — Greek mode names with zero context. "Ionian = Major scale" and "Aeolian = Natural Minor" are never mentioned.
- **Box positions**: No explanation of what a "box" is, why there are 5, or which to practice first.
- **No Finger display mode**: Scales show Interval and Note but not Finger. Finger guidance would be useful for learning left-hand patterns.

---

### Progressions Screen (`ProgressionsScreen.tsx`)

**Strengths:**
- Chord quality colour-coding on the diatonic row (gold = Major, blue = Minor, red = Dim) is intuitive once you understand the domain.
- Hint text "Tap the circle to change key" is present.

**Issues:**
- **Roman numerals**: I, ii, iii, IV, V, vi, vii° are never explained. The degree symbol (°) looks like a formatting error to a beginner.
- **"Diatonic"**: The word is used in the UI and the hint text but never defined.
- **Circle of Fifths**: A complex music theory visualisation with three concentric rings (major keys, relative minors, leading-tone diminished). The relationship between tapping the circle and the diatonic chord row above it is never explained. No legend for what each ring represents.
- **Key concept**: "Tap the circle to change key" — but "key" itself is never defined.
- **No chord names in the diatonic row**: Beginners think in C, Dm, G — not I, ii, V. No option to show chord names alongside numerals.

---

### Triads Screen (`TriadsScreen.tsx`)

**Issues:**
- **"Triads"**: Never explained. (A triad is a 3-note chord: root, third, fifth.)
- **"Inversion"**: Never explained. "1st Inv" and "2nd Inv" are opaque labels.
- **String group picker**: Shows "1-2-3", "2-3-4", "3-4-5", "4-5-6" — no explanation of why you'd restrict to a string group or what a "string group" is.
- **"Dim" / "Aug"**: Abbreviations for diminished and augmented — fine for musicians, opaque for beginners.
- **Why separate from Chords?**: Nothing in the UI explains the conceptual distinction between Chords and Triads.

---

### Arpeggios Screen (`ArpeggiosScreen.tsx`)

**Issues:**
- **"Arpeggios"**: Never explained. (Playing chord tones one note at a time.)
- **Sweep order numbers in Finger mode**: Numbers 1, 2, 3, … indicate the order to play notes for sweep-picking. This is a useful feature but completely unexplained. No hint text.
- **No audio**: An arpeggio is inherently about sequence and timing — neither can be conveyed without sound.

## Component-Level Findings

### `GuitarNeck.tsx`

- **Colour legend absent**: Five distinct dot colours with no legend anywhere in the UI.
- **Interactivity not signalled**: In the Chords screen, tapping a root dot changes the voicing — but there is no visual affordance (no glow, border, or cursor) to indicate these dots are tappable.
- **String naming**: Shows string names (E A D G B E) but does not indicate which end is high vs. low, or that the top string in standard view is the thinnest.
- **Strengths**: Left-hand mode, capo bar visualisation, note dimming behind capo, realistic string width variation — all solid.

### `TypePicker.tsx`

- Horizontal scroll with no end indicator — users may not discover all chord/scale types available.
- No tooltips or glossary links on any label.

### `ChordDiagram.tsx`

- Standard format (circles, crosses, open circles) — but the legend for reading a chord diagram (what ×, ○, and ● mean) is never shown.
- Rendered at 90×112 px — small on modern screens.

### `ScalePositionPicker.tsx`

- Fret range shown ("Box 1 (1–4)") — good.
- No explanation of what a box position is or why 5 exist.

## Quick Wins (Small Changes, High Impact)

1. **Add text labels under tab icons** — "Chords", "Scales", "Progressions", "Triads", "Arpeggios". Costs 30 minutes, eliminates the guessing game for every new user.

2. **Add a colour legend below the fretboard** — A static or dynamic row showing what each dot colour represents in the current display mode. One hour, massive clarity improvement.

3. **Add hint text to every screen** — Each screen already has a hint area. Populate all of them:
   - Scales: "Select a box position to focus on a specific neck region."
   - Triads: "Triads are 3-note chords. Inversions reorder the notes — same chord, different shape."
   - Arpeggios: "Numbers show the order to play each note (sweep-picking direction)."
   - Progressions: "These 7 chords all fit together in the key of [X]. Tap any chord to add it to your progression."

4. **Improve capo label** — Replace "C" with "Capo" or a clamp icon, and increase tap target size. The TODO already tracks this.

5. **Show chord names alongside Roman numerals in Progressions** — Add a small secondary label under each numeral button (e.g., "I → C", "ii → Dm"). Makes the screen immediately useful to a beginner without removing the theory layer.

6. **Rename ambiguous abbreviations**:
   - "Nat. Minor" → "Natural Minor"
   - "Harm. Minor" → "Harmonic Minor"
   - "Dim" → "Diminished" (or "Dim (Diminished)")
   - "Aug" → "Augmented" (or "Aug (Augmented)")

7. **Add a "?" button next to the Circle of Fifths title** — Opens a modal explaining what the three rings represent and how tapping the circle changes the key.

## Deeper Improvements

### 1. Onboarding

The app drops users into a feature-rich screen with no guidance. A minimal onboarding (2–3 screens shown once on first launch) would reduce immediate abandonment:

- Screen 1: "Tap a root note and chord type to see where to play on your guitar."
- Screen 2: "Switch between Finger, Interval, and Note views to see different information."
- Screen 3: "Explore Scales, Triads, Arpeggios, and Progressions in the tabs below."

Alternatively, a single welcome banner with a "Dismiss" button would cost less but still orient new users.

### 2. Glossary / Contextual Help

A modal glossary (accessible via a "?" icon in the header) covering at minimum: Interval, Triad, Arpeggio, Inversion, Mode, Diatonic, Voicing, Capo, Chord Degree, Pentatonic. Each entry needs only 1–2 sentences and possibly a visual example.

### 3. Beginner Mode

A toggle in Settings (future) that simplifies each screen:
- Chords: show only Major, Minor, 7th — hide Sus, Dim7, Min7b5 etc.
- Scales: show only Major, Natural Minor, Major Pent., Minor Pent. — hide Harmonic, Melodic, modes.
- Progressions: show chord names (C, Dm, G) instead of Roman numerals.

### 4. Progressions Tab — Chord Name Labels

The biggest single comprehension barrier on the screen is the Roman numeral system. Showing "I (C)", "ii (Dm)", "iii (Em)" under each button would make the screen usable for beginners immediately, with no loss for intermediate users.

### 5. Fretboard Interactivity Affordance

In the Chords screen, root dots are tappable to change voicings — but nothing signals this. A subtle ring or glow on interactive dots, plus updated hint text, would make this discoverable.

### 6. Tab Order — Learning Logic

Current order: Chords → Scales → Progressions → Triads → Arpeggios.

A more logical beginner order would be: Chords → Scales → Triads → Arpeggios → Progressions. The current order places the most theory-heavy tab (Progressions) at position 3, before the more foundational Triads and Arpeggios. Consider reordering or at minimum adding a difficulty indicator.

### 7. Settings Screen

Move LH, ♯/♭, and theme into a dedicated Settings screen behind a gear icon. The header is already crowded and will only get worse. Capo arguably stays (it's a session-level control), but everything else should move. This is the same item in the TODO's "Dependent on a prior decision" section.

## Summary

The app is technically polished — the fretboard rendering is excellent, the music theory engine is accurate, and features like left-hand mode and capo visualisation are well-implemented. The UX gap is almost entirely in communication: the app knows a lot but tells the user very little.

The fastest path to a significantly better beginner experience is:

1. Add text labels under the tab icons.
2. Add a fretboard colour legend.
3. Add hint text to every screen (most screens already have a hint area).
4. Show chord names alongside Roman numerals in Progressions.
5. Fix the capo label (already tracked in TODO).

These five changes require no architectural work, touch only UI text and layout, and would dramatically reduce the confusion a new user experiences in the first 60 seconds.
